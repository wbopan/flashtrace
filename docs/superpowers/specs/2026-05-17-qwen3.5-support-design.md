# FlashTrace 对 Qwen3.5 系列模型的支持 — 设计文档

日期: 2026-05-17
分支: `worktree-qwen3.5-support`

## 1. 背景与目标

FlashTrace 的 IFR（Information Flow Routes）归因方法完全建立在**每层 softmax 注意力权重**之上：
`flashtrace/core.py` 对每一层都假设一个 Llama/Qwen 风格的解码块，含 `self_attn.{q,k,v,o}_proj`、
统一的 GQA 头数，并通过 `output_attentions` 或从 Q/K 重算得到 `[H,S,S]` 注意力矩阵。

**Qwen3.5-9B 不满足这个假设。** 已从 HuggingFace `config.json` 确认其真实架构：

- `model_type: qwen3_5`，架构 `Qwen3_5ForConditionalGeneration`（多模态，含 vision tower）。
- 文本解码器 32 层，`layer_types` 为 `3×linear_attention + 1×full_attention` 循环
  （`full_attention_interval: 4`）—— 即 **24 层 Gated DeltaNet（线性注意力）+ 8 层 full attention**。
- `full_attention` 层：16 Q 头 / 4 KV 头，`head_dim=256`，`attn_output_gate=true`，
  partial RoPE（`partial_rotary_factor=0.25`）+ 交错 mRoPE。
- `linear_attention` 层（Gated DeltaNet）：`linear_num_key_heads=16`、`linear_num_value_heads=32`、
  key/value head dim 128、`linear_conv_kernel_dim=4`。
- 需要 `transformers >= 5.2.0`（qwen3_5 建模代码自 5.2.0 起进入 transformers；当前环境为 4.53.1）。

**目标**：让 FlashTrace IFR 在 Qwen3.5-9B 上产出归因，且在本项目的 smoke test 中对同一批样本
达到「忠实度接近 Qwen3-8B 且高于 perturbation 基线」。

## 2. 关键设计决策（已与用户确认）

1. **线性注意力层的归因**：对每个 Gated DeltaNet 层**展开 delta-rule 递推**，构造等价的下三角
   `S×S` token 混合矩阵 `A`（满足 `attn_out = A @ V`），喂给 IFR 的 V/O proximity 机制。
   这是最忠实的方案，也是达到「接近 qwen3」的最优路径。
2. **对照基准**：smoke test 中的 qwen3 对照模型用 **Qwen3-8B**（与 9B 同档，公平对比）。

## 3. 架构设计

### 组件 A — 环境隔离
worktree 独立 uv 环境升级 `transformers>=5.2.0`，修复 FlashTrace 对 transformers 5.x 的兼容性
（既有合成模型测试套件作为回归基线）。

### 组件 B — 解码器发现（`extract_model_metadata`）
泛化解码器栈定位：从 model 向下遍历属性，找到含 decoder block `ModuleList`（`.layers`）的模块，
以支持 VLM 的嵌套结构（`Qwen3_5ForConditionalGeneration` 的文本栈嵌套在 language model 内）。

### 组件 C — 每层几何
`IFRParameters` 当前持有全局 `n_heads_q/n_kv_heads/head_dim/group_size`。重构为**每层** `LayerGeometry`。
`ModelMetadata` 改为携带逐层类型描述符：`FullAttentionLayer` / `LinearAttentionLayer`，
各自带几何与权重引用。`core.py` 的 IFR 主循环改为读取每层几何。

### 组件 D — 等效注意力 provider
对每层产出 `(attn_matrix [H,S,S], v_weight, o_weight, geometry)`：
- **full attention 层**：`attn` 取自 `output_attentions`；v/o 取 `self_attn.{v,o}_proj`。
- **linear attention 层**：经 hook 捕获 DeltaNet 子模块的 `q,k,v,β,decay`，用 chunked/WY 展开
  构造逐头下三角 `A`（`A = (I - tril(diag(β)KKᵀ,-1))⁻¹ diag(β)` 与衰减组合），使 `attn_out = A@V`；
  v/o 取 DeltaNet 的 value / output 投影。smoke test 序列短，`O(S²)` 展开可行。

### 组件 E — 门控（gating）
`attn_output_gate`（full）与 DeltaNet 输出门控均为注意力后逐 token/通道的缩放。
首版可先省略门控、测量忠实度；若未过基准线再将捕获的门控值并入 IFR 贡献。

### 组件 F — smoke test（`tests/test_qwen35_support.py`）
- **(a) 合成微模型**：用 `AutoConfig` 构造少层、linear+full 混合的 tiny `qwen3_5` 文本模型，
  跑 generate→trace，断言每 token 分数有限、形状正确。作为不需 18GB 下载的开发回路。
- **(b) 真实模型对照**（脚本 / 慢测标记）：在 N 个固定样本上，对 Qwen3.5-9B 跑 FlashTrace
  与 perturbation 基线、对 Qwen3-8B 跑 FlashTrace，用本项目忠实度 AUC 指标
  （`flashtrace/improved.py: faithfulness_test_skip_tokens`）评估，断言
  `flashtrace(qwen3.5) ≥ perturbation(qwen3.5)` 且 `flashtrace(qwen3.5) ≈ flashtrace(qwen3-8b)`。

## 4. 成功标准
1. 既有合成模型测试套件在 transformers 5.x 下全绿（回归不破）。
2. 合成 qwen3_5 微模型的 generate→trace smoke test 通过。
3. 真实对照：Qwen3.5-9B 上 FlashTrace 忠实度 ≥ perturbation 基线，且与 Qwen3-8B 接近。

## 5. 范围外
- 多模态（图像/视频）输入归因 —— 仅支持文本 token 归因。
- Qwen3.5 MoE 变体（`qwen3_5_moe`）的专门优化 —— 解码器发现应能容纳，但不专门调优。
